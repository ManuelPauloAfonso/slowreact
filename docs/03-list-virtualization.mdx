## What is happening

This component renders a list of 1,000 items using a standard `.map()`. The
browser mounts every DOM node at once including the ones that are completely
off-screen. Scroll through the list and open the Performance tab in Chrome
DevTools to see the impact.

## Hints

- The browser renders every item in the list, even the ones you can't see
- List virtualization only renders the items visible in the viewport
- `react-window` handles this without scroll jank or memory leaks
- Only virtualize if profiling confirms the list is the bottleneck

## The problem

```tsx
interface Item {
  id: number
  name: string
}

interface Props {
  items: Item[]
}

export default function ListVirtualization({ items }: Props) {
  return (
    <ul>
      {items.map((item) => (
        <li key={item.id} style={{ height: 50 }}>
          {item.name}
        </li>
      ))}
    </ul>
  )
}
```
